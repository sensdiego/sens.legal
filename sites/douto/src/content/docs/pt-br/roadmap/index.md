# Roadmap

O roadmap do Douto agora parte de uma definicao simples:

**Douto e o fornecedor interno de doutrina do Jude.md para o Valter.**

## Direcao Consolidada

- consumidor primario: Valter;
- autoridade inicial: contratos e processo civil;
- retrieval e sintese permanecem no escopo;
- qualidade vem antes de expansao;
- markdown/Obsidian fica como camada editorial, nao como centro do produto.

## Estado Atual

- existe pipeline local;
- existe busca CLI local;
- nao existe entrega programatica madura;
- nao existe gate confiavel para liberar sintese.

## Sequencia Oficial

1. Fundacao reproduzivel
2. Quality gate
3. Contrato de entrega ao Valter
4. Retrieval explicavel
5. Sintese com gate proprio

## Backlog Canonico

O backlog canonico do Douto esta no Linear em `repo/douto`, de `SEN-447` a `SEN-468`, com `blockedBy` impondo ordem de execucao.

## Trabalho Deliberadamente Adiado

- ranking hierarquico e exploracoes parecidas ficam depois da linha base (`SEN-446`)
- qualquer interface tempo real ou MCP direto fica atras da entrega boa ao Valter
